21 What is the difference between HashMap & HashTable?
→ HashMap introduced in java 1.2
→  None of the methods in Hashmap are  synchronized and hashmap is the most commonly used Map  implementation in java
→ We can store only null key but any number of null values
 --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
 → HashTable introduced in java 1.0, so it is legacy.
→ All the methods in HashTable are synchronous so only one thread can access HashTable at a timeHashTable will not any null keys, if we are trying to insert null key we will nullpointer exception.
22  How ConcurrentHashMap works internally?
Concurrent HashMap  will internally use bucket level locking to avoid concurrent modification exception  whereas HashTable or synchronized collections will use object level locking. So only one thread can access  HashTable Or SynchronizedHashMap but in ConcurrentHashMap at a time 16 threads can access one object.          
23 FailFast  vs FailSafe?
When we use ArrayL ist OR HashMap in multithreaded environment we may get ConcurrentModification exception So we call ArrayList/HashMap as FastFail.The concurrent classes example CopyOnArrayList or ConcurrentHashMap are example for FailSafe.             Fail Fast will throw Concurrent Modification Exception, but FailSafe will never throw Concurrent Modification Exception.
24 What is ConcurrentModificationException?
When we try to modify a collection on multi threaded environment every thread has a local midcount variable, if that variable value is mismatched with Collection Midcount variable value then it will throw ConcurrentModificaitonException.
25 Comparable vs Comparator ?
Comparable interface present in java.lang package and it has only one abstract method which is compareTo method takes only one parameter. String  and all the predefined wrapper classes which are already implemented Comparable interface.      
Comparator is an interface which is present on java.util package which has only one abstract method compare which takes 2 parameters. In java none of the predefined classes implement Comparator.We can say Comparable for natural sorting order and Comparator for Custom sorting order.
26 How to avoid ConcurrentModificationException in a multi-threaded Environment?
To avoid Concurrent Modification Exception we can declare Collections in local variables or we can use concurrent collections like CopyOnWriteArrayList or Concurrent HashMap.

28 What is Executor Framework in java?
 When we create a thread using Runnable interface or extend a Thread class those are non re-usable  threads. As we can call the Start method only once, if we are trying to call one more start method one more time, we will get an Illegal state exception.  So, to create re-usable threads in java we can use ExecutorService Framework which was introduced in java 1.5. We can call the newFixedThreadPool method on the Executors class to create an ExecutorService   threadpool. By using this executor service we can execute any number of tasks.
29 Callable vs Runable?
Runnable is present in the java.lang package and it has only one method run() which does not take any parameter and it won't return anything. And Runnable has been present since java 1.0 onwards. 
Callable is present in java.util.concurrent package and has only method which is call() which does not take any parameter but it can return some value. Callable is available from java 1.5
30 Difference between String Buffer and String Builder?
String Buffer was introduced in java 1.2 and every method string buffer is synchronized so only one thread can access String Buffer Object at a time.
StringBuilder was introduced in java 1.5 and none of the methods are synchronized so at a time multiple Threads can access one StringBuilder object. 
31 What is Immutable class?
Once the object is created if we are trying to change the state of the object with that modified state there is a new Object created. 
32 What is String Constant Pool Memory Area (OR) Why String is Immutable class In java
In the JVM for String littlers there is separate memory available which is the String Constant Pool area. 
For Example: String s1 = “Sunil”;  String s2 = “Sunil”;   String s3 = “Sunil”;          
It looks like we created 3 objects but only one String object created in String Constant Pool memory and the same object will be shared across multiple references . So, changing this value from any reference variable will impact on other references, so to solve this issue Java made String as an Immutable  class. 
34 Difference between Vector and ArrayList?
 Vector is introduced in java 1.0 and ArrayList is introduced in java 1.2 and every method in  Vector is synchronized but none of the methods in ArrayList are  synchronized. So, all multiple threads can access Arraylist at a time but Vector can accessed one object at a time.
36 What is Singleton design pattern?
One object per JVM is called singleton design pattern in java. By default in java we can create any number of objects for a class so we need to write extra logic to make it a single object.         
 →   Make the constructor as private           
 →  Write a static factory method to return the same object again and again 
37 What is Serialization in java?
The process of converting an object into network transferal mode is called Serialization. For that in java the class needs to implement a Serializable interface. 
38 What is functional interface?
Functional interface is an interface which has exactly one abstract method but can contain any number of default and static methods. In Java we can write lambda expressions only for functional interfaces. Even though it is optional to annotate  functional interface with @FunctionalInterfce annotation it is highly recommended to use that to stop to added any new abstract methods in Functional interface.
39 What is the root parent for Exception hierarchy?
Throwable is the root parent for exception hierarchy.
40 How to create Custom Exception?
Write a java class and extends it from either Exception or RuntimeException. If we extends from Exception class it is checked Exception and if it is extended from RuntimeException then it is Un-Checked Exception.
