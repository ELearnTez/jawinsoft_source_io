41  Methods in Object Class? 
1) public final Class getClass()   - Returns the Class class object of this object. The Class class can further be used to get the metadata.   
2) public int hashCode() - Returns the hashcode number for this object.            
3) public boolean equals(Object obj)  - Compares the given object to this object.  
4) protected Object clone() throws CloneNotSupportedException  - Creates and returns the exact copy (clone) of this object.                    5) public String toString()  - R eturns the string representation of this object.      
6) public final void notify() - Wakes up single thread, waiting on this object's monitor. 
7) public final void notifyAll() - Wakes up all the threads, waiting on this object's monitor.      
8) public final void wait(long timeout)throws InterruptedException  - Causes the current thread to wait for the specified milliseconds, until another thread notifies (invokes notify() or notifyAll() method).
9) public final void wait(long timeout,int nanos)throws InterruptedException
causes the current thread to wait for the specified milliseconds and nanoseconds, until another thread notifies (invokes notify() or notifyAll() method).
42 Overloading vs Overriding?
→  Overloading or static polymorphism, when there are 2 methods  having the same name but different   parameters is called overloading.    → Override is a process when we want to change the parent class method in child class. There are some rules we need to follow to override a method       
→ The scope of the method should be the same or increased but not decreased. Example, if parent class method scope is default then in child class method scope should default or protected or public but should not be private.
→ The return type of the method should be the same co-variant type.
→The name of the method should be same.
→The number parameters should be same
→ If the parent class method throws some checked exception, then the child class should throw the same checked exception or its child type but not the parent exception.
43 What is class loader and How many classloader is there in java?
In Java Classloader is of three types:
 i) BootStrap ClassLoader: A Bootstrap Classloader is a Machine code which kickstarts the operation when      the JVM calls it. It is not a java class. Its job is to load the first pure Java ClassLoader. Bootstrap ClassLoader loads classes from the location rt.jar. Bootstrap ClassLoader doesn’t have any parent ClassLoaders. It is also called as the Primodial ClassLoader.           
ii) Extension ClassLoader: The Extension ClassLoader is a child of Bootstrap ClassLoader and loads the extensions of core java classes from the respective JDK Extension library. It loads files from jre/lib/ext directory or any other directory pointed by the system property java.ext.dirs.            
iii) System ClassLoader: An Application ClassLoader is also known as a System ClassLoader. It loads the        Application type classes found in the environment variable CLASSPATH, -classpath or -cp command line option. The Application ClassLoader is a child class of Extension ClassLoader.

45 What Stream API?
Java provides a new additional package in Java 8 called java.util.stream. This package consists of classes, interfaces and enum to allows functional-style operations on the elements.  With Stream api we can process collections very effectively.  A stream is not a data structure instead it takes input from the Collections, Arrays or I/O channels.
Streams don’t change the original data structure, they only provide the result as per the pipelined methods.               Each intermediate operation is lazily executed and returns a stream as a result, hence various intermediate operations can be pipelined. Terminal operations mark the end of the stream and return the result.
46 What is intermediate operators and terminal operators in Streams?
Stream is introduced in Java 8, it is only used for processing group of data not for the storting elements.. It does not modify the actual collection, they only provide the result as per the pipelined methods.Stream api supports multiple operations and operations are divided into two parts −
Intermediate Operation- These operations are used to pipeline other methods and to transform into the other streams. They don’t produce results because these operations are not invoked until the terminal operation gets executed. Filter, map, sorted, peek, distinct methods are examples for intermediate operators.          
Terminal operations - These operations are used to produce results. They can’t be used for chaining the other methods. forEach, count, toArray methods are examples for terminal operators.
47  What is difference between parallel stream and normal stream?
Stream() method will return a Stream object which will run by only one thread. For example , if we have a list of products then all products will be processed by only one thread.
Parallelstream() will return stream objects which will run multiple objects at a time since it will internally multi-thread mechanism.  For example , if we have a list of products then every product will be processed by a separate thread so we process data parally.

55 Access Modifiers in java?
There are 4 access modifiers in java private  , <default>, protected, public             
a) private is java reserved keyword which is applicable at variable & method level. If a variable or method is declared as private we can not be able to access that from outside that class.            
b) <default> - class , variables and method, If a class or variable or methods is declared as <default> then those can be accessed within the package. from outside of the package we will not be able to call those classes / variables/ methods.              
c) Protected :  if a variable or method is declared as protected we can access them within the package and from outside of the package through child classes we can access.            
d) public: If a class or variable or method is declared as public we can access them from anywhere.
