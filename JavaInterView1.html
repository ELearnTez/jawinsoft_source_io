1  What Java 8 Features you used in your project?

Default & Static Methods
 In InterfacesUntil Java 1.8 interfaces are pure abstract but from 1.8 onwards interfaces can not be pure abstract, it can have some implementation through default and static methods.
Functional Interfaces
An interface which has exactly one abstract method is called Functional Interface. Functional Interface will have only one abstract method but can have any number of Default and static methods
Lambda Expression
Only for functional interfaces we can write lambda expression and through lambda expression we can write functional programming in java
Stream API
This is Java 1.8 new feature. We use Stream api to process Collections more effectively.  With the help of functional interfaces and lambda expressions and Stream api we can process collections with very less lines of code. 
Optional Class: 
Instead of writing if else conditions to check if an object has data or not we can use Optional class. Optional class has methods like isPresent, ifPresent, orElse, orElseThrow to write functional programming to avoid to write if else  to check data present or not

9  What are the most commonly used collections in your project?
 ArrayList is the most commonly used collection in my project. ArrayList is to store individual elements with duplicate elements are allowed and null insertion is possible and insertion order is preserved. ArrayList is the best performer when we insert and retrieve the data. ArrayList is a bad performer when we delete or insert data in the middle of an array list. We use Linked List when we insert or delete the data in the middle of arrayList. We use HashSet when we store individual elements and duplicates are not allowed and insertion order is not required. When we want to store the data in Key Value pair format we will use Map implementations example: HashMap When we want non duplicates and with some sorted order we will use TreeSet or TreeMap
10 What is diffrence between ArrayList and LinkedList?
When we want to store and retrieve the data then ArrayList is sufficient but if we are deleting or adding data from the middle of arraylist, then arraylist will perform worst as ArrayList has to perform multiple adjacent operations which will make arraylist slow. So, when we want to delete or add an element in the middle of the arraylist then we have to use LinkedList.
11 What is the difference between Checked Exception and Un- Checked Exception?
 a) Checked Exception
 The classes that directly inherit the Throwable class except RuntimeException and Error are known as checked exceptions. For example, IOException, SQLException, etc. Checked exceptions are checked at compile-time.    
b) Unchecked Exception
 The classes that inherit the RuntimeException are known as unchecked exceptions. For example, ArithmeticException, NullPointerException, ArrayIndexOutOfBoundsException, etc. Unchecked exceptions are not checked at compile-time, but they are checked at runtime.
12 Difference between Exception and Error?
Both are child classes of Throwable but Exceptions are recoverable and Errors are unrecoverable. Exceptions will occur due to application issues and Errors will occur due to Hardware failures.
13 Diffrence between  throw and throws?
In java throw and throws both are java reserved keywords. We use throw keyword to throw exceptions inside the method body. throws is the keyword which we use at method declaration level to delegate the exception handling responsibility to the caller of the method.

18 OOPS Concepts In Java?
a) Abstraction : We use abstraction to hide the implementation from the outside world . We can achieve abstraction using interfaces and abstract classes in Java.   
b) Inheritance :  To acquire parent class properties into child class we implement Inheritance. And We can  achieve Inheritance using extends & implements keywords in java.1) class can implement any interfaces 2) an interface can extend any number interfaces3) a class can extend only one class 4) a class can extend only one class & any number interfaces.
c) PloyMorphism:
i) static Polymorphism   / overloading
ii) dynamic Polymorphism  / Overriding             
Static Polymorphism  / Overloading : We can achieve static polymorphism with Inheritance        without Inheritance. The method name is the same and with different parameters are called overloading or static polymorphism. The method resolution happens at compile time that is the reason why we call it a static polymorphism.
 Dynamic Polymorphism/Method Overriding : When we want to modify the parent class method in child class we can override that method. When we want to override a method then that method signature must be the same as parent class method. To make sure we are properly overriding the parent class method, we can add @Override annotation on top of the child class method. 
 --> To override a method we need a parent child relationship. Where inheritance is required for method   overriding.
 --> We can't override final methods        
 --> We can't override private methods,since private method scope is within the class. So. parent class private methods should not be  visible to child class                
--> We Can't override static methods. Since, static methods are stored in the method area not in the Heap     area. So, for method area override concept not applicable
 --> For Variables & Constructors the overriding concept is not applicable.       
D) Encapsulation:- To provide security to our data we write tightly encapsulated class. In Java when a class, all properties should be private and public setters & getters to set and get the data. 
19 What is the contract b/w hashcode and equals method?
-> When hashcode is the same of 2 objects which both objects will store in the same bucket, then the equals method might return true or f alse.
-> W hen equals method returns true for 2 objects, then hashcode method must return same value for those 2 objects.( the object are equals must be stored in same bucket)
-> If hashcode is not the same for 2 objects, equals methods will always return false.
->  If the equals methods return false, then hashcode might be the same or different for 2 objects.
20 How HashMap works internally?
When we store or retrieve the data in HashMap it uses hashcode and equals method. When we create HashMap the default size of hashmap is 16 which means there are 16 buckets created. In which bucket the data will be stored will identify the by using hashcode method, once the bucket is identified  As Hashmap will not allow duplicates keys it will execute equals method to check whether that object is present or not. So, Hashmap internally use hashcode and equals method to store and retrieve the data.
